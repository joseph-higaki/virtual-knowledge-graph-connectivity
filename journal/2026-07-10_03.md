## Decisions (and why)
- **Refactor before rung 4.** Author `polyglot.obda` + loaders once against final column names and
  store placement, not twice. Column rename done against a green rung 2 for an isolated checkpoint.
- **CbG ‚Üí Iceberg (both compound edges in the lake with `compound`).** Provenance-coherent ‚Äî one
  source system owns the compound entity and its edges. Keeps the green rungs untouched, still forces
  cross-catalog on both new edges (they join to PG `gene`/`disease`), and makes DaG a clean
  intra-Postgres edge. The user's "one entity proprietary / many in lakehouse" framing was refined to
  provenance/system-of-record, not entity count.
- **`load_postgres` DROP+CREATE** (was CREATE IF NOT EXISTS + TRUNCATE): a column/schema change must
  land on a pre-existing `pgdata` volume; TRUNCATE kept the stale columns and broke COPY. Symmetric
  with `load_iceberg`.
- **Discovered the ground-truth predicates** (`hetio:binds`, `hetio:treats`) by querying GraphDB, not
  assuming ‚Äî the exact metaedge verbs, so label parity holds across both endpoints.
- **`explain.py` parses catalogs from `EXPLAIN (TYPE IO, FORMAT JSON)` of Ontop's *own* reformulated
  SQL** ‚Äî a faithful DoD proof (Trino did the federation), not a hand-written stand-in query.
- **Query files decoupled from rung/topology.** The serving path is a property of the run, not the
  query; the UI (whose job *is* to show the serving path) names `Ontop ‚Üí Trino`.
