## Decisions (and why)
- **Full README TBox, incl. `rdfs:domain`/`rdfs:range` — user's call over a flagged constraint.** Hard
  constraint #1 says "no OWL reasoning; stop and flag." Loading domain/range *does* engage Ontop's type
  inference, so I asked; the user chose the richer schema over the vocabulary-only alternative. Kept it
  because it's provably parity-neutral here (below).
- **Why it stays parity-neutral.** Two compounding reasons: (a) `SELECT DISTINCT` collapses a gene reached
  via both its node row and an edge, so `COUNT(?gene)` doesn't inflate; (b) the slice has **referential
  integrity** — every edge endpoint also exists as a node — so the domain/range branches introduce no IRI
  the mappings didn't already type. If that invariant ever broke, those branches are where Ontop would
  diverge from the materialized ground truth first.
- **`v0.1.2`, not `v0.2.0`.** Additive capability (schema now served) with zero behavior change to rungs,
  mappings, loaders, or parity results. Patch semantics, consistent with the v0.1.1 precedent.
