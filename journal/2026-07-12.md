## Decisions (and why)
- **`v0.1.1`, not `v0.2.0`.** Everything since `v0.1.0` is docs, diagrams, UI, and comment rewording —
  zero change to rungs, mappings, loaders, or parity logic (query diffs are header-comment-only). Patch
  semantics.
- **README states final state; rungs live in `docs/staggered-execution.md`.** The README is the "what it
  is" record; the staggered build order is execution detail. Splitting them keeps the front page about the
  architecture, not the ladder.
- **Drug Lake diagram excludes Trino.** Trino is the federation *engine* that reads the lake, not part of
  the storage source; modeling it inside "Drug Lake" would misrepresent the boundary the whole harness
  exists to demonstrate.
- **Vendored UI libs, not CDN.** Keeps the compare UI runnable offline and CSP-safe; versions pinned in
  `VERSIONS.txt`, consistent with the repo's pin-in-code rule.
- **One catch-up journal row for the whole docs/UI phase.** The INDEX exists for cumulative build cost that
  sums cleanly; ~9 short un-journaled sessions are recorded as one Session 05 row (~8.2M) rather than lost
  or scattered.
